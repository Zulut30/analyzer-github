# RepoScope Design System

## Направление

RepoScope использует стиль Liquid Glass: полупрозрачные поверхности, сильный blur, мягкие внутренние блики, тонкие стеклянные грани и спокойные цветовые акценты. Интерфейс должен ощущаться как легкая рабочая панель поверх светящейся среды, а не как набор плотных карточек.

## Основные принципы

- Первое действие всегда в фокусе: строка поиска GitHub-репозитория должна быть самой заметной интерактивной зоной.
- Стекло строится слоями: прозрачный fill, верхний блик, нижняя грань, цветной glow и backdrop blur.
- Навигация и меню выглядят как плавающие стеклянные капсулы.
- Текст остается контрастным во всех темах; декоративные блики не должны ухудшать читаемость.
- На мобильных экранах меню не должно требовать горизонтальной прокрутки.

## Токены

- `--glass`: базовая прозрачная поверхность.
- `--glass-strong`: более плотная стеклянная поверхность для важных контролов.
- `--glass-muted`: тихий стеклянный фон для вторичных панелей.
- `--glass-border`: светлая грань стекла.
- `--glass-highlight`: верхний блик.
- `--accent`, `--blue`, `--amber`, `--coral`: цветовые акценты и glow.

## Header

Header фиксируется сверху как отдельная стеклянная капсула. Он состоит из:

- бренд-блока с иконкой RepoScope;
- центральной навигации с иконками;
- кнопки выбора темы;
- кнопки профиля с аватаром и статусом.

Header должен иметь:

- `backdrop-filter: blur(...) saturate(...)`;
- тонкую светлую рамку;
- внутренний верхний highlight;
- нижнюю световую линию;
- мягкую тень, отделяющую панель от фона.

## Theme Menu

Меню темы открывается как стеклянный popover с сеткой 3x3.

Темы:

- Светлая
- Темная
- Сумерки
- Нефрит
- Индиго
- Розовый
- Янтарь
- Океан
- Фиолетовая

Выбранная тема подсвечивается синей рамкой и внешним свечением. Swatch должен показывать настроение темы, а не быть плоским цветным квадратом.

## Profile Menu

Профильное меню показывает:

- аватар GitHub;
- имя или состояние входа;
- статус доступа к анализу;
- быстрые ссылки на профиль, историю и избранное;
- действие входа или выхода.

Если пользователь не вошел, меню остается полезным: объясняет, какой GitHub-профиль нужен для доступа.

## Search Bar

Строка поиска должна выглядеть как самая стеклянная часть интерфейса.

Обязательные слои:

- прозрачная капсула с blur;
- верхний диагональный блик;
- нижняя тонкая световая грань;
- внутреннее затемнение снизу;
- цветной glow при focus;
- отдельная стеклянная кнопка действия.

На мобильных экранах кнопка поиска превращается в квадратную icon-only кнопку.

## Responsive

- Desktop: header в одну строку, nav по центру, действия справа.
- Tablet/mobile: бренд и действия в первой строке, nav ниже.
- Small mobile: nav раскладывается в 2x2, без горизонтальной прокрутки.
- Popover не должен выходить за viewport.

## Performance

- Не добавлять тяжелые изображения ради декоративного стекла.
- Предпочитать CSS gradients, pseudo-elements и existing bitmap assets.
- Не отправлять UI-метрики, историю, тему или GitHub activity в AI-промпт анализа.
- Сохранять blur только на крупных ключевых поверхностях: header, menus, search, cards.

## Accessibility

- Все меню должны иметь `aria-expanded`, `aria-controls` и закрываться по Escape.
- Активная тема должна иметь `aria-pressed`.
- Focus states должны быть видимыми на всех темах.
- Текст не должен полагаться только на цвет или blur.
